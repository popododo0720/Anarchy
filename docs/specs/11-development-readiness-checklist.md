# Anarchy Development Readiness Checklist

Status: implementation gate

Development can begin when all are true:
- product baseline doc exists
- architecture doc exists
- development rules doc exists
- API/communication doc exists
- CLI-first plan exists
- Helm/GitOps docs exist
- core resource model exists
- network/IP spec exists
- storage spec exists
- code layout spec exists
- implementation phases doc exists
- lab plan is clear

Before each feature begins:
- feature/domain spec exists
- CLI behavior spec exists if needed
- external API spec exists if needed
- proto contract exists if needed
- tests are identified
